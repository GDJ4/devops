Ship a static site with nginx and rsync

A compact static page you can sync to a remote host in seconds. Keep the layout simple, the files small, and deployments repeatable.

View deploy steps rsync docs

Why this stack

Static files, stable delivery

Fast sync

rsync transfers only the changed bytes, so updating the server stays quick even on slow links.

Predictable serving

nginx serves files straight from disk with caching headers you control. No runtime surprises.

One command deploy

The provided script pushes site/ to the server path you choose and can reload nginx for you.

How to ship

Deploy in three steps

  1. 1
    Install nginx on the server. Point its root to your deploy path (example in infra/nginx.static-site.conf).
  2. 2
    Set SSH access. Ensure your key is allowed on the server; keep REMOTE_HOST, REMOTE_USER, and REMOTE_PATH ready.
  3. 3
    Run ./deploy.sh. The script syncs site/ to the server and can reload nginx.
REMOTE_HOST=203.0.113.10 \
REMOTE_USER=deploy \
REMOTE_PATH=/var/www/static-site \
./deploy.sh
